## Resumen ejecutivo
- Se crearon 4 rutas API nuevas + 1 método (PATCH) + 1 validador compartido, reutilizando `airtable-client.ts` y `make-client.ts` (sin crear helpers nuevos).
- Endpoints de escritura degradan con `pendiente_make: true` porque los escenarios Make (SC-Asignar, SC-Edicion) aún no están provisionados.
- `pnpm tsc --noEmit` limpio (exit 0).

## Decisiones técnicas
- **Reuse:** `postToMake` (HMAC ya implementado) y `getRecord/listRecords/createRecord` en vez de los `lib/airtable/client.ts` / `lib/make/webhook.ts` que proponía el plan §3.2.
- **Contrato de error REGLA B** centralizado en `issuesToCampos()` (aplana `path` de zod a `unidades.1.campo`) → 422 `{error:'validacion', campos:[...]}`. Conflictos de negocio → 409 `{error:'conflicto_negocio', campo, motivo}`.
- **Idempotencia REGLA A:** `asignar` relee `tasador`/`estado` del registro real antes de reenviar; 409 si ya hay tasador o si estado es terminal.
- **REGLA C server-side:** `PATCH` relee `estado`; 409 si ≠ `creada`.
- **Degradación honesta:** si falta el webhook Make, se responde `{ok:true, pendiente_make:true}` (200) + `console.warn`, en vez de fabricar un éxito silencioso o crashear (patrón de `crear-solicitud`).

## Overrides aplicados (rutas reales vs plan)
- Plan §3.1 pedía `POST /api/solicitudes` para crear → se mantiene el existente `POST /api/webhooks/crear-solicitud` (no se duplicó).
- Plan usaba `AIRTABLE_API_KEY` → el repo usa `AIRTABLE_TOKEN` (env real). Sin cambios en el nombre.
- Plan pedía `lib/airtable/client.ts` + `lib/make/webhook.ts` → ya existen como `lib/airtable-client.ts` + `lib/make-client.ts`; se apuntó a ellos.
- **Expansión de filtros de `GET /api/solicitudes`** (q, orden, prioridad, tasador, page, pageSize) **diferida a P5**, donde se reconcilia el enum `Vista` — hacerla ahora obligaría a rehacerla al reconciliar vistas.

## Bugs / obstáculos y resolución
- Tipo del issue de zod 4: `z.core.$ZodIssue` es frágil entre versiones. Se tipó el helper con un subconjunto estructural `{ path: PropertyKey[]; message: string }[]` → robusto y `tsc` limpio.
