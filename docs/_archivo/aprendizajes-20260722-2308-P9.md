## Reglas nuevas → MIGRAR a docs/aprendizajes.md
- E-P9-1: Para generar un blueprint Make, extraer el vocabulario de módulos de un blueprint REAL del mismo repo/instancia (module id + version + estructura del mapper) en vez de inventarlo; validar el JSON con `node -e JSON.parse` antes de darlo por importable. Si no hay referencia para un módulo (p. ej. email), dejarlo como TODO manual, no fabricarlo — un módulo inexistente rompe todo el import.
- E-P9-2: Los scripts `lint`/`test`/`format` del `package.json` v0 son aspiracionales (eslint/vitest/playwright/prettier no instalados). El gate de calidad real del repo es `pnpm build` + `pnpm tsc --noEmit`. Instalar el tooling es una tanda aparte.
- E-P9-3: Un health check útil valida dependencias externas (lectura mínima a Airtable), no sólo devuelve 200 fijo — así Railway detecta un token roto o base caída.
